# Security Policy

## Reporting a Vulnerability

**DO NOT open a public issue for security bugs.**

Report via one of:

1. **Email**: you@example.com (PGP key on request)
2. **GitHub Private Vulnerability Reporting**: https://github.com/your-github-username/stellar-split/security/advisories/new

## Our commitments

- Acknowledge within **48 hours**
- Provide a fix timeline within **7 days**
- Credit reporters in release notes (unless they prefer anonymity)
- Disclose publicly after fix is released

## Scope

**In scope:**
- Code in `/packages`, `/apps`, `/contracts`
- CI/CD workflows
- Build & release pipeline

**Out of scope:**
- Third-party dependencies (report upstream)
- Social engineering
- DOS attacks
- Issues requiring physical access

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 1.x     | ✅        |
| < 1.0   | ❌        |

## Bounty

We don't currently offer paid bounties but will publicly acknowledge all valid reports in our Hall of Fame.
